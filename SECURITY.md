# Security Policy

- Signalement : ouvrir une issue en la taguant "security" sans inclure de secrets ni données sensibles. Fournir l’environnement (`APP_ENV`), les versions et étapes de reproduction.
- Communication : si des détails sensibles sont nécessaires, demandez un canal privé dans l’issue.
- Corrections : proposer des PR sans divulguer de secrets ; les secrets doivent rester gérés via SOPS/age ou variables d’environnement.
